# 异构模型调度策略

## 核心原则
根据任务复杂度选择模型等级，整体算力成本控制在全员顶配模型的1/2~1/5。

## 模型等级划分

### 高级模型（高成本、高质量）
适用场景：
- 架构设计、跨模块冲突决策
- 复杂业务逻辑、数据库架构
- 安全漏洞扫描、风险审计
- 需求梳理、业务规则建模
- 前后端联调验证

### 标准模型（低成本、快速响应）
适用场景：
- CRUD代码生成、简单配置文件
- 页面布局、UI组件、样式
- 测试脚本生成、CI配置
- 访谈纪要整理、文档格式化
- 任务派单、文档整理

## 各角色调度建议

| 角色 | 高级模型 | 标准模型 |
|------|---------|---------|
| A 架构师 | 架构设计、冲突决策 | 配置生成、文档整理 |
| B 售前 | 需求建模、竞品分析 | 纪要整理、流程图 |
| C 后端 | 复杂业务、DB设计 | CRUD、工具类 |
| D 前端 | 状态联动、联调 | 页面、组件、样式 |
| E 工程 | 安全审计、测试策略 | 测试脚本、CI配置 |
